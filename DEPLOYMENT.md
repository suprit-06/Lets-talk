# Let's Talk Deployment Guide

## Step 6: Deployment Instructions

### 1. Railway Deployment (Recommended)
Railway makes it incredibly easy to deploy FastAPI and PostgreSQL.

1. Create a Railway account and click **New Project**.
2. Provision a **PostgreSQL** database from the Railway dashboard.
3. Connect your GitHub repository containing this code.
4. Add the following **Environment Variables** in the Railway service settings:
   - `GROQ_API_KEY`: Your free Groq API key
   - `SECRET_KEY`: Generate a random secure string
   - `DATABASE_URL`: (Will be automatically provided by Railway, just ensure it uses `postgresql+asyncpg://` instead of `postgresql://`)
5. Railway will automatically detect the `Dockerfile` and build/deploy your application.
6. To run database migrations, you can add a start command: `python -m app.init_db && uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### 2. Render Deployment
1. Go to Render.com and create a **New Web Service**.
2. Connect your GitHub repository.
3. Set the Environment to **Docker**.
4. Create a new **PostgreSQL** database on Render and note the connection string.
5. In the Web Service environment variables:
   - Add `DATABASE_URL` (change `postgres://` to `postgresql+asyncpg://`)
   - Add `GROQ_API_KEY`
   - Add `SECRET_KEY`
6. Deploy. Render will build the Docker container and host it.

## Step 7: Local Development Guide

1. **Python Environment**:
```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Mac/Linux
source venv/bin/activate
```

2. **Install Dependencies**:
```bash
pip install -r requirements.txt
```

3. **Database Setup**:
- Install PostgreSQL locally or run via Docker:
  ```bash
  docker run --name ai-chat-db -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres
  ```
- Copy `.env.example` to `.env` and fill in `GROQ_API_KEY` and the `DATABASE_URL`.

4. **Initialize Database**:
```bash
python -m app.init_db
```

5. **Run the Server**:
```bash
uvicorn app.main:app --reload
```
App will be running at [http://localhost:8000](http://localhost:8000)

## Step 8: Production Scaling Suggestions

1. **Gunicorn with Uvicorn Workers**:
   In production, don't just run `uvicorn` directly. Instead, update the Dockerfile CMD to use Gunicorn to manage multiple workers:
   ```dockerfile
   CMD ["gunicorn", "app.main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000"]
   ```

2. **Database Connection Pooling**:
   For high traffic, place **PgBouncer** in front of PostgreSQL to manage the massive number of concurrent connections generated by FastAPI's async nature.

3. **Redis Caching**:
   If users request the same system prompts or static responses, cache them in Redis. You can also use Redis for more robust Rate Limiting across multiple horizontally scaled containers.

4. **CDN for Static Assets**:
   Host the Bootstrap CSS, JS, and font files on a CDN (like Cloudflare or AWS CloudFront) rather than serving them directly from the FastAPI server via `/static`. This reduces server load and speeds up page paints.
